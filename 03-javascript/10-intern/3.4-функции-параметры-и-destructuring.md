# 3.4 Функции, параметры и destructuring

Функции в JavaScript — не просто «кусок кода, который можно вызвать». Через них мы задаем интерфейс поведения: какие данные принимает логика, что возвращает и насколько предсказуемо это читать.

## 🎯 Что ты освоишь

- function declaration, function expression и arrow function;

- destructuring в сигнатуре и теле функции;

- где краткость помогает, а где уже мешает.

---

## Что важно понять

- Выбор формы функции влияет не только на стиль записи, но и на поведение `this`, hoisting и читаемость.
- Параметры — это контракт функции, а не просто удобный синтаксис.
- Хорошая функция на intern-уровне делает одну вещь понятно, а не демонстрирует весь синтаксис языка разом.

---

## Термины по-человечески

- `Function declaration` — объявление функции, доступное до места вызова в пределах hoisting-правил.
- `Function expression` — функция как значение, присвоенное переменной.
- `Arrow function` — компактная форма функции со своим поведением относительно `this`.
- `Destructuring` — извлечение нужных частей из массива или объекта в отдельные переменные.

---

## Базовый пример

```js
function formatPrice(value, { currency = 'RUB', locale = 'ru-RU' } = {}) {
  return new Intl.NumberFormat(locale, {
    style: 'currency',
    currency,
  }).format(value);
}
```

---

## Частые ошибки

- Используют arrow function по умолчанию, не понимая разницу по `this`.
- Смешивают много параметров без структуры и имен.
- Деструктурируют слишком агрессивно и теряют читаемость сигнатуры.
- Прячут сложную логику в короткую запись ради компактности.

---

## Anti-pattern

"Раз можно записать в одну строку, значит так и надо".

Для функций важнее ясный контракт и читаемость, чем минимальное число символов.

---

## 🔧 Практика

1. Перепиши одну длинную функцию на более ясную сигнатуру.
1. Используй параметры по умолчанию и объект-опции там, где аргументов стало много.
1. Сравни обычную функцию и arrow function на примере с `this`.
