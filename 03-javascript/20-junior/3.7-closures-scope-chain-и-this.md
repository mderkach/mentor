# 3.7 Closures, scope chain и this

Это одна из первых тем, где JavaScript перестает быть «язык с простым синтаксисом» и начинает требовать модели выполнения в голове. Большая часть странного поведения функций сидит именно здесь.

- переменные и scope из [3.1 Переменные, hoisting и temporal dead zone](../10-intern/3.1-переменные-hoisting-и-temporal-dead-zone.md).

## 🎯 Что ты освоишь

- что такое closure без мистики;
- как работает scope chain;
- откуда берется `this` в разных формах вызова;
- зачем нужны `call`, `apply`, `bind`;
- почему часть JS-багов выглядит как «значение потерялось» или «контекст неожиданно другой».

---

## Что важно понять

- `Closure` — это не продвинутая техника сама по себе, а естественное следствие того, как функции видят внешние переменные.
- `this` задается не местом объявления, а способом вызова, за исключением arrow function.
- Хороший уровень на этой теме — уметь объяснить конкретный runtime-кейс, а не повторить определение из статьи.

---

## Термины по-человечески

- `Closure` — функция помнит доступ к внешнему scope даже после завершения внешней функции.
- `Scope chain` — цепочка областей видимости, по которой движок ищет имя переменной.
- `this` — текущий контекст вызова функции, а не «владелец функции».

---

## Базовый пример

```js
function createCounter() {
  let count = 0;

  return () => {
    count += 1;
    return count;
  };
}

const next = createCounter();
next(); // 1
next(); // 2
```

---

## Частые ошибки

- Думают, что `this` работает одинаково во всех функциях.
- Теряют контекст метода при передаче его как callback.
- Считают closure чем-то «тяжелым» без понимания, где он реально полезен или опасен.
- Лечат все подряд через `.bind(this)`, не разбираясь в причине.

---

## Anti-pattern

"Если `this` ведет себя странно, просто завернем всё в arrow function".

Иногда это помогает, но без понимания модели вызова ты просто меняешь поведение вслепую.

---

## 🔧 Практика

1. Разбери 5 примеров с обычной функцией, методом объекта и arrow function.
1. До запуска предскажи значение `this` в каждом.
1. Напиши маленький factory через closure и объясни, почему состояние сохраняется.
