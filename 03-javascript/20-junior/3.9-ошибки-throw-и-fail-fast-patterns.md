# 3.9 Ошибки, throw и fail-fast patterns

JavaScript-код без внятной работы с ошибками быстро превращается в систему, где сбой то молча проглатывается, то валит весь поток внезапно. Этот урок про предсказуемое поведение при нештатных сценариях.

## 🎯 Что ты освоишь

- `throw`, `try/catch/finally`;
- `Error` и custom error classes;
- error bubbling;
- assertions и fail-fast patterns;
- как не прятать ошибку под «мягкое» поведение без причины.

---

## Что важно понять

- Ошибка — это тоже часть контракта функции и модуля.
- Не каждая проблема должна «тихо переживаться»: иногда система должна падать раньше и яснее.
- Хороший junior-код либо обрабатывает ошибку по делу, либо честно пробрасывает ее выше.

---

## Термины по-человечески

- `Throw` — явное выбрасывание ошибки.
- `Bubble up` — подъем ошибки по стеку вызовов вверх.
- `Fail-fast` — ломаем выполнение раньше, чем испорченные данные уйдут глубже в систему.

---

## Базовый пример

```js
function getUserName(user) {
  if (!user) {
    throw new Error('User is required');
  }

  return user.name ?? 'Anonymous';
}
```

---

## Частые ошибки

- Глотают ошибку в `catch` без логики восстановления.
- Возвращают `null` и `Error` вперемешку без контракта.
- Используют `throw 'string'`, а не `Error`.
- Пытаются «не пугать пользователя» и скрывают реальную проблему от системы и разработчика.

---

## Anti-pattern

"Добавим `try/catch`, и код станет надежнее".

Сам по себе `catch` не делает систему устойчивее, если он просто прячет проблему.

---

## 🔧 Практика

1. Возьми функцию без валидации входа и добавь fail-fast поведение.
1. Перепиши строковый `throw` на нормальный `Error`.
1. Разбери, где ошибку нужно обработать локально, а где пробросить выше.
