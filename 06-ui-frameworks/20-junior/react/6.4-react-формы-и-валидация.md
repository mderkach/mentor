## Термины урока

- `Form state` — это значения полей, ошибки, touched/dirty-состояния и фаза submit.
- `Controlled field` — это поле, чье текущее значение хранится в framework state.
- `Uncontrolled field` — это поле, где текущее значение живет в DOM, а framework читает его по мере необходимости.
- `Schema validation` — это проверка данных по явной схеме.
- `Field array` — это повторяющийся набор полей вроде списка телефонов или адресов.
- `Server error mapping` — это привязка ошибок сервера к конкретным полям или общему состоянию формы.

## Реализация в React

В React формы особенно быстро упираются в цену controlled-state на каждый символ. Поэтому `React Hook Form` часто полезен как pragmatic default: он умеет работать через `ref`, уменьшает лишние ререндеры и дает понятный form lifecycle.

```tsx
import { useForm } from "react-hook-form";
import { z } from "zod";
import { zodResolver } from "@hookform/resolvers/zod";

const signupSchema = z.object({
  email: z.string().email("Нужен корректный email"),
  password: z.string().min(8, "Минимум 8 символов"),
});

type SignupFormValues = z.infer<typeof signupSchema>;

function SignupForm() {
  const {
    register,
    handleSubmit,
    formState: { errors, isSubmitting },
  } = useForm<SignupFormValues>({
    resolver: zodResolver(signupSchema),
  });

  return (
    <form onSubmit={handleSubmit(async (values) => save(values))}>
      <input type="email" {...register("email")} />
      {errors.email && <p>{errors.email.message}</p>}

      <input type="password" {...register("password")} />
      {errors.password && <p>{errors.password.message}</p>}

      <button type="submit" disabled={isSubmitting}>
        Создать аккаунт
      </button>
    </form>
  );
}
```

Границы здесь такие:

- HTML отвечает за семантику поля и native affordances;
- React/form library отвечает за UI flow и form state;
- schema отвечает за структуру данных и правила;
- сервер отвечает за финальную валидацию и бизнес-правду.

`Controller` нужен там, где поле не умеет жить через обычный `register`.
`useFieldArray` нужен там, где форма реально динамическая.

---

## Частые ошибки

- держат большое form state на десятках `useState`;
- дублируют validation rules в UI и schema без причины;
- используют client validation как замену server validation;
- тянут `Controller` туда, где обычный input уже работает;
- забывают про touched/dirty/submit lifecycle и начинают показывать ошибки слишком рано.

---

## Anti-pattern

"Форма — это обычный local state плюс пара `if` на submit".

На маленьком примере это прокатывает. На продуктовой форме быстро превращается в долговую яму.

---

## 🔧 Практика

1. Возьми одну форму и разложи ее на значения, touched/dirty, submit lifecycle, client validation и server errors.
1. Определи, где там controlled-state действительно нужен, а где можно жить проще.
1. Найди один сценарий, где без field array или нормальной schema validation форма станет хрупкой.
1. Придумай, как эта же задача выглядела бы во Vue через composables и form library.
