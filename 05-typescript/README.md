# 🔷 5. TypeScript

В этом разделе ты изучаешь TypeScript: типы, функции, интерфейсы, обобщения, настройку компилятора и работу с данными в приложении.

Здесь разбираются:

- базовые типы, функции и состояния;
- моделирование данных из API и внутри приложения;
- `interface`, `class` и `abstract class` без путаницы в их назначении;
- `generics`, `utility types`, `narrowing` и другие инструменты типизации;
- `tsconfig`, модули, декларации и сборка;
- проверка входных данных и работа с внешними интеграциями;
- типы на границах пакетов, библиотек и больших проектов.

Важно: TypeScript не заменяет JavaScript и сам по себе не проверяет внешние данные. Если ты не понимаешь, что реально происходит в коде во время выполнения, типы быстро превращаются в `as any`.

## Что такое TypeScript и какие проблемы он реально решает

`TypeScript` — это надстройка над JavaScript со статической системой типов и компилятором, который проверяет код до запуска и превращает его в обычный JavaScript.

На практике он нужен не ради "строгих аннотаций", а ради вполне приземлённых проблем:

- рассинхрон между API, UI и внутренними моделями;
- небезопасный рефакторинг, когда поле или тип поменяли не везде;
- слепая работа с `any`, `undefined` и состояниями "здесь вроде точно есть данные";
- слабая читаемость границ между слоями и пакетами.

Что TypeScript не чинит сам:

- не проверяет внешние данные сам по себе;
- не исправляет плохую модель данных;
- не заменяет понимание JavaScript, модулей, `class`, `this` и реального поведения объектов.

Короткая формула для всего раздела:

- JavaScript отвечает за то, что реально произойдёт в браузере.
- TypeScript помогает заранее увидеть, где модель данных и код расходятся с реальностью.

## Как учиться в этом разделе

- Идти по уровням: **Стажер -> Junior -> Middle -> Senior -> Senior+**.
- Не учить TypeScript как список конструкций. Каждый урок должен отвечать на вопрос: какую практическую проблему он помогает решить.
- Сначала собрать базу по типам и моделированию данных, потом переходить к модулям, конфигу компилятора и сильным темам.
- Если проблема упирается в реальное поведение модулей, `class`, `this` или асинхронности, возвращаться в [3. JavaScript](../03-javascript/README.md).
- Если тема уходит в сборку, алиасы, монорепу и пайплайн проекта, подключать [7. Infrastructure](../07-infrastructure/README.md).
- Держать в голове простое ограничение: типы не работают во время выполнения. Поэтому на сильных уровнях важно понимать, где одной типизации уже недостаточно.
