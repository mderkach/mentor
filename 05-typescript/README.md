## 🎯 Как проходить раздел

1. Выбери 1-2 компетенции под текущие задачи проекта.
2. Закрой ключевые навыки по порядку: понимание -> применение -> проверка.
3. Закрепи результат на практических задачах из [TASKS.md](../TASKS.md).

## 📈 Ожидания по уровням

| Уровень | Ожидания |
|---|---|
| Intern | Использует базовые типы, interfaces и type aliases. |
| Junior | Работает с unions, generics, utility types и type guards в прикладном коде. |
| Middle | Моделирует сложные контракты через mapped/conditional types, project references и strict config. |
| Senior | Проектирует типобезопасные API, shared contracts, runtime schemas и типовую архитектуру модулей. |
| Architect | Определяет TS-стратегию, rules of modeling и правила evolvability для большой кодовой базы. |
