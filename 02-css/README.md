- Если база по HTML еще плавает, сначала вернись в [1. HTML](../01-html/README.md).
- Внутри CSS-слоя иди последовательно: `Нулевой блок -> Стажер -> Джун -> Мидл -> Сеньор`.

## Как учиться в этом разделе

- Идём по уровням: **Нулевой блок -> Стажер -> Джун -> Мидл -> Сеньор**.
- В каждом уроке: объяснение, правила выбора инструментов, практика.
- Цель: не просто "знаю свойство", а умею строить устойчивый CSS без каскадного хаоса, accessibility-долга и архитектурного разъезда.
- Если плавает целостная картина браузерного пайплайна, сначала пройди: [4.0 Браузер: от URL до пикселя](../04-dom-и-browser/30-middle/4.0-браузер-от-url-до-пикселя-network-fetch-render.md).

## ✅ Ожидания по уровням

| Уровень | Что должен уметь |
|---|---|
| Нулевой блок | Уверенно использовать DevTools для диагностики CSS-конфликтов до правок в коде. |
| Стажер | Понимать каскад, селекторы, box model, position и уверенно собирать простые layout на flex. |
| Джун | Делать production-вёрстку: grid, типографика, адаптив, эффекты и анимации без ломания UX. |
| Мидл | Осознанно внедрять modern CSS, токены, accessibility/system preferences, сложные scroll/layer-сценарии и styling strategy в продуктовой архитектуре. |
| Сеньор / Senior+ | Формировать CSS-стандарты команды: слои каскада, quality gates, governance, theming strategy и правила масштабирования между приложениями и командами. |
