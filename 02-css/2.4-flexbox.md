# 2.4 Flexbox

Flexbox — основной инструмент для одномерных layout: ряд или колонка, выравнивание, распределение пространства.

---

## 🎯 Что ты освоишь

- как работает ось `main` и `cross`;
- как выравнивать элементы без костылей;
- где `flex` идеален, а где лучше `grid`;
- почему `min-width: 0` иногда спасает интерфейс.

---

## База: контейнер и элементы

```css
.list {
  display: flex;
  gap: 12px;
  align-items: center;
}
```

Ключевые свойства контейнера:
- `flex-direction`;
- `justify-content`;
- `align-items`;
- `flex-wrap`;
- `gap`.

Ключевые свойства элемента:
- `flex-grow`;
- `flex-shrink`;
- `flex-basis`;
- `align-self`.

---

## Когда использовать flex

Лучшие кейсы:
- navbar;
- строка кнопок;
- карточка "контент + действия";
- вертикальное выравнивание в небольших компонентах.

Когда лучше не flex:
- сложная двумерная сетка с контролем и по строкам, и по колонкам -> это `grid`.

---

## Частый баг: текст не сжимается

Если внутри flex-элемента длинный текст, он может выталкивать layout.

Решение:

```css
.item {
  min-width: 0;
}

.item__title {
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
}
```

---

## Паттерн "липкий футер"

```css
.page {
  min-height: 100dvh;
  display: flex;
  flex-direction: column;
}

.page__main {
  flex: 1;
}
```

Этот подход надёжнее, чем хаки с абсолютным позиционированием футера.

---

## Частые ошибки

- Путают `justify-content` и `align-items`.
- Используют `margin` вместо `gap` для всех случаев.
- Не включают `wrap`, когда элементы должны переноситься.
- Пытаются строить полноценную сетку через вложенные flex-контейнеры.

---

## 🔧 Практика

1. Собери header: логотип слева, меню по центру, кнопка справа.
2. Сделай ряд карточек с переносом на новую строку через `flex-wrap`.
3. Исправь layout с длинным заголовком через `min-width: 0`.
