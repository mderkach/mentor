# 2.3 Поток документа, блочная модель и позиционирование

Большинство "непонятных" багов в верстке — это не магия, а проблемы потока, box model и неправильного `position`.

---

## 🎯 Что ты освоишь

- как работает normal flow;
- как считать размеры блока через box model;
- когда использовать `relative`, `absolute`, `fixed`, `sticky`;
- как не ломать layout из-за `z-index` и переполнения.

---

## Поток документа

По умолчанию элементы идут сверху вниз:
- блочные (`div`, `section`) занимают всю строку;
- строчные (`span`) живут внутри строки.

Если резко менять `position` или `float` без понимания потока, верстка "плывёт".

---

## Box model без сюрпризов

Размер блока = `content + padding + border + margin`.

С `box-sizing: border-box` проще:
- заданная `width` уже включает `padding` и `border`;
- меньше сюрпризов на адаптиве.

Полезно всегда проверять в DevTools вкладку `Computed`, если блок "не влез".

---

## Позиционирование: что когда

- `position: relative` — база для смещения и точка отсчёта для `absolute`-потомков.
- `position: absolute` — для бейджей, иконок, локальных оверлеев внутри конкретного блока.
- `position: fixed` — элементы, привязанные к viewport (например, floating action button).
- `position: sticky` — "прилипание" в пределах родительского контейнера.

Пример:

```css
.card {
  position: relative;
}

.card__badge {
  position: absolute;
  top: 8px;
  right: 8px;
}

.sidebar {
  position: sticky;
  top: 16px;
}
```

---

## `z-index` и контексты наложения

`z-index` работает только в своём stacking context.

Что часто ломает ожидания:
- `transform`, `opacity < 1`, `position` + `z-index` у предков могут создать новый контекст;
- поэтому "z-index: 9999" не всегда спасает.

---

## Overflow и скролл

- `overflow: hidden` может обрезать тени/всплывашки;
- `overflow: auto` даёт локальный скролл;
- глобальный горизонтальный скролл обычно означает проблему ширин/отступов.

---

## Частые ошибки

- Пытаются выровнять layout через `position: absolute` вместо flex/grid.
- Используют `fixed`, когда нужен `sticky`.
- Не понимают, от какого блока считается `absolute`.
- Лечат наложение через бесконечный рост `z-index`.

---

## 🔧 Практика

1. Сделай карточку с бейджем через `relative + absolute`.
2. Сделай колонку фильтров через `position: sticky`.
3. Найди баг с наложением слоёв и исправь через корректный stacking context.
