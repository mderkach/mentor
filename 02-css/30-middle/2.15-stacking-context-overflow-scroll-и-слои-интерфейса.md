# 2.15 Stacking context, overflow, scroll и слои интерфейса

Это один из самых частых production pain point в CSS.
Интерфейс визуально "почти работает", но tooltip обрезается, sticky не липнет, fixed ведет себя странно, а `z-index: 9999` внезапно ничего не решает.

---

- `stacking context`, `z-index`, clipping и containing block;
- где `overflow` создает локальный мир со своими проблемами;
- как `transform` конфликтует с `fixed` и `sticky`;
- `scroll-snap`, nested scroll containers и управление скроллом;
- как перестать лечить слои интерфейса магическими числами.

## Термины по-человечески

- `Stacking context` — локальный мир наложения слоев, внутри которого `z-index` сравнивается только со "своими".
- `Clipping` — ситуация, когда родитель физически обрезает содержимое по своей области.
- `Containing block` — блок, от которого считаются координаты и размеры позиционированного потомка.
- `Nested scroll container` — внутренний контейнер со своим скроллом внутри уже скроллящейся страницы.
- `Scroll snapping` — принудительная фиксация прокрутки на заранее заданных точках.

---

## Рабочий пример

```css
.layout {
  display: grid;
  grid-template-columns: 280px minmax(0, 1fr);
  gap: 24px;
}

.sidebar {
  position: sticky;
  top: 16px;
}

.panel {
  overflow: auto;
  max-block-size: 60vh;
}

.carousel {
  display: grid;
  grid-auto-flow: column;
  overflow-x: auto;
  scroll-snap-type: x mandatory;
}

.carousel > * {
  scroll-snap-align: start;
}
```

---

## Что важно понять

- `z-index` не абсолютная сила, а локальная договоренность внутри stacking context.
- `overflow: hidden|auto|scroll` часто не просто чинит визуал, а меняет поведение слоев, sticky и tooltip/popup.
- `transform` у предка может создать новый containing block и неожиданно повлиять на `fixed`/`sticky`.
- nested scroll containers быстро портят UX: колесо мыши, touch scroll и keyboard navigation начинают жить в нескольких "мирах".
- scroll snapping полезен, когда он помогает навигации, а не воюет с пользователем.

---

## Частые production-сценарии

### Tooltip или dropdown обрезается

Чаще всего виноваты:

- `overflow: hidden` у родителя;
- новый stacking context у предка;
- popup рендерится внутри контейнера, где ему уже тесно.

### Sticky "не работает"

Чаще всего виноваты:

- неправильный scroll-контейнер;
- родитель с `overflow`, который меняет контекст;
- отсутствие явного `top`;
- sticky-элемент выше контейнера, чем может "прилипнуть".

### Fixed ведет себя как будто не fixed

Проверь, не появился ли сверху `transform`, `filter` или другой эффект, который создал новый containing block.

---

## Частые ошибки

- лечат баг слоя только ростом `z-index`;
- не проверяют `overflow` и clipping у предков;
- создают nested scroll containers без явной UX-причины;
- используют sticky внутри контекста, где он технически не может сработать как ожидалось.

---

## Decision framework

- Если проблема в наложении, сначала ищи stacking context, а не повышай `z-index`.
- Если проблема во всплывашке, сначала проверяй clipping и scroll container.
- Если проблема в sticky/fixed, сначала проверяй containing block и `overflow`.
- Если внутри страницы уже есть один локальный скролл, каждый следующий добавляй только с очень понятной причиной.

---

## Anti-pattern

"Не видно popup? Поставим `z-index: 9999`".

Если родитель уже создал отдельный stacking context или обрезал содержимое, число чаще всего не спасет.

---

## 🔧 Практика

1. Возьми экран с sidebar, popup и длинным списком.
1. Найди все места, где появляются локальные scroll-контейнеры.
1. Проверь, какие предки создают stacking context.
1. Исправь один баг без увеличения `z-index` и один баг без удаления `overflow` "вслепую".
