# 2.10 Современный CSS 2024-2026

Современный CSS уже закрывает задачи, ради которых раньше тянули JS или тяжёлые UI-надстройки. Важно не просто знать фичи, а внедрять их безопасно.

---

## Что уже должно быть знакомо

- каскад, специфичность и базовые селекторы из [2.1 Базовые знания CSS](../10-intern/2.1-базовые-знания-css.md), [2.2 Селекторы, псевдоклассы и псевдоэлементы](../10-intern/2.2-селекторы-псевдоклассы-и-псевдоэлементы.md);
- адаптивность и units из [2.9 Адаптивность, responsive и units](../20-junior/2.9-адаптивность-responsive-и-units.md).

---

## 🎯 Что ты освоишь

- `@layer` для управляемого каскада;
- CSS Nesting без перегруза селекторов;
- container queries для компонентов;
- `:has()` для условных состояний;
- `@scope` и `@property` для изоляции и typed-переменных;
- `light-dark()`, `env()`, `min/max/clamp`, `attr()`;
- progressive enhancement через `@supports`.

---

## Термины урока

- `Progressive enhancement` — это подход, где новая CSS-фича улучшает UI, но без неё интерфейс остаётся рабочим.
- `Container query` — это условие стилей, зависящее от размера контейнера компонента, а не всего viewport.
- `Cascade layer` — это слой каскада с фиксированным приоритетом (`@layer`), который снижает конфликты специфичности.

---

## Что важно понять

- Modern CSS полезен не сам по себе, а как способ уменьшить JS-зависимость, каскадный долг и избыточные abstractions.
- Новые фичи нельзя внедрять как «коллекцию модных приемов»: у каждой должна быть задача, fallback и границы применимости.
- Middle-уровень здесь начинается там, где ты уже не просто знаешь синтаксис, а умеешь безопасно встраивать его в продукт.

---

## `@layer`: порядок стилей как контракт

```css
@layer reset, base, components, utilities;

@layer base {
  body {
    margin: 0;
  }
}

@layer components {
  .button {
    border-radius: 12px;
  }
}
```

Что это даёт:

- предсказуемый каскад;
- меньше "войн" специфичности;
- проще ревью и масштабирование дизайн-системы.

Когда слои становятся системным правилом команды, это уже архитектурный уровень. Больше о внедрении на масштабе: [2.17 CSS-архитектура команды: слои, соглашения, масштабирование](../40-senior/2.17-CSS-архитектура-команды-слои-соглашения-масштабирование.md).

## Nesting

```css
.card {
  padding: 16px;

  &__title {
    font-weight: 700;
  }

  &:hover {
    box-shadow: 0 8px 20px rgba(15, 23, 42, 0.12);
  }
}
```

Правило: nesting нужен для читаемости, а не для 5 уровней вложенности.

Если проект на SCSS, удерживай такие же ограничения на вложенность. Больше о SCSS-правилах: [2.11 Препроцессор SCSS/SASS](./2.11-препроцессор-scss-sass.md).

## Container queries

Вместо зависимости от viewport компонент реагирует на размер своего контейнера.

```css
.card-list {
  container-type: inline-size;
}

@container (min-width: 640px) {
  .card-list {
    grid-template-columns: repeat(2, minmax(0, 1fr));
  }
}
```

Идеально для переиспользуемых виджетов.

Пиши безопасный базовый вариант без `@container`, а потом улучшай его контейнерными условиями. Для junior-базы по units и viewport сначала полезно пройти: [2.9 Адаптивность, responsive и units](../20-junior/2.9-адаптивность-responsive-и-units.md).

## `:has()`

```css
.field:has(input:invalid) {
  border-color: #dc2626;
}
```

Позволяет стилизовать родителя на основе состояния потомка.

Используй аккуратно и проверяй поддержку для целевых браузеров.

---

## `@scope`: локальная область стилей

```css
@scope (.checkout-widget) {
  .title {
    margin-block-end: 0.75rem;
  }
}
```

`@scope` помогает ограничить влияние стилей конкретным контейнером. Это снижает риск побочных эффектов в больших приложениях.

---

## `@property`: типизированные custom properties

```css
@property --progress {
  syntax: '<number>';
  inherits: false;
  initial-value: 0;
}

.meter {
  --progress: 0.75;
  opacity: var(--progress);
}
```

Это полезно, когда переменная участвует в анимации и тебе нужна предсказуемая типизация. Подробно про токены и runtime-переменные: [2.12 CSS Custom Properties и Design Tokens](./2.12-CSS-custom-properties-и-design-tokens.md).

---

## Современные функции значений

```css
.panel {
  padding: max(1rem, env(safe-area-inset-left));
  border-radius: clamp(0.75rem, 0.5rem + 1vw, 1.25rem);
  color: light-dark(#0f172a, #e2e8f0);
}
```

- `min()/max()/clamp()` упрощают адаптив без лишних брейкпоинтов;
- `env()` полезен для safe-area на мобильных;
- `light-dark()` подбирает значение под текущую цветовую схему;
- `attr()` можно использовать для ограниченных CSS-сценариев, но проверяй поддержку.

---

## Что ещё знать из modern CSS

- `anchor-positioning` — перспективный способ позиционировать popover/tooltip от якоря (используй с fallback);
- `scroll-snap-type` и `scroll-snap-align` — управляемый snapping в каруселях и лентах;
- View Transitions API и `::view-transition*` — аккуратные межсостоянийные переходы между UI-сценами;
- `appearance`, `user-select`, `touch-action`, `pointer-events`, `cursor`, `resize` — свойства "поведения", которые часто решают UX-точечные задачи без JS.

Если хочется отдельно собрать в голове сложные сценарии `scroll`, `overflow`, clipping и слои интерфейса, дальше есть отдельный модуль: [2.15 Stacking context, overflow, scroll и слои интерфейса](./2.15-stacking-context-overflow-scroll-и-слои-интерфейса.md).

## Progressive enhancement

```css
.card {
  border-radius: 12px;
}

@supports (text-wrap: balance) {
  .card__title {
    text-wrap: balance;
  }
}
```

Смысл: современная фича улучшает UI, но без неё всё должно оставаться рабочим.

---

## Частые ошибки

- Внедряют новую фичу без проверки поддержки и fallback.
- Используют nesting как копию SCSS-хаоса.
- Не фиксируют порядок каскада, а потом снова воюют со специфичностью.
- Пытаются решить всё одной "модной" фичей.

---

## Anti-pattern

"Раз браузеры уже поддерживают фичу, значит можно просто начать писать на ней весь проект".

Поддержка фичи еще не равна безопасному внедрению в production.

---

## 🔧 Практика

1. Раздели стили проекта на слои через `@layer`.
1. Добавь container query для карточного компонента.
1. Внедри одну modern CSS-фичу через `@supports` с fallback.
