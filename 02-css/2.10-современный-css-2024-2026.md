# 2.10 Современный CSS 2024-2026

Современный CSS уже закрывает задачи, ради которых раньше тянули JS или тяжёлые UI-надстройки. Важно не просто знать фичи, а внедрять их безопасно.

---

## 🎯 Что ты освоишь

- `@layer` для управляемого каскада;
- CSS Nesting без перегруза селекторов;
- container queries для компонентов;
- `:has()` для условных состояний;
- progressive enhancement через `@supports`.

---

## `@layer`: порядок стилей как контракт

```css
@layer reset, base, components, utilities;

@layer base {
  body {
    margin: 0;
  }
}

@layer components {
  .button {
    border-radius: 12px;
  }
}
```

Что это даёт:
- предсказуемый каскад;
- меньше "войн" специфичности;
- проще ревью и масштабирование дизайн-системы.

---

## Nesting

```css
.card {
  padding: 16px;

  &__title {
    font-weight: 700;
  }

  &:hover {
    box-shadow: 0 8px 20px rgba(15, 23, 42, 0.12);
  }
}
```

Правило: nesting нужен для читаемости, а не для 5 уровней вложенности.

---

## Container queries

Вместо зависимости от viewport компонент реагирует на размер своего контейнера.

```css
.card-list {
  container-type: inline-size;
}

@container (min-width: 640px) {
  .card-list {
    grid-template-columns: repeat(2, minmax(0, 1fr));
  }
}
```

Идеально для переиспользуемых виджетов.

---

## `:has()`

```css
.field:has(input:invalid) {
  border-color: #dc2626;
}
```

Позволяет стилизовать родителя на основе состояния потомка.

Используй аккуратно и проверяй поддержку для целевых браузеров.

---

## Progressive enhancement

```css
.card {
  border-radius: 12px;
}

@supports (text-wrap: balance) {
  .card__title {
    text-wrap: balance;
  }
}
```

Смысл: современная фича улучшает UI, но без неё всё должно оставаться рабочим.

---

## Частые ошибки

- Внедряют новую фичу без проверки поддержки и fallback.
- Используют nesting как копию SCSS-хаоса.
- Не фиксируют порядок каскада, а потом снова воюют со специфичностью.
- Пытаются решить всё одной "модной" фичей.

---

## 🔧 Практика

1. Раздели стили проекта на слои через `@layer`.
2. Добавь container query для карточного компонента.
3. Внедри одну modern CSS-фичу через `@supports` с fallback.
