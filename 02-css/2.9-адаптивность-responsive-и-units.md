# 2.9 Адаптивность, responsive и units

Адаптивность — это не "ужмём десктоп до мобильного", а проектирование интерфейса под разные устройства и сценарии.

---

## 🎯 Что ты освоишь

- mobile-first подход;
- выбор единиц измерения (`px`, `rem`, `%`, `vw`, `dvh`);
- медиазапросы без хаоса;
- `clamp()` для гибкой типографики и отступов.

---

## Mobile-first

Пиши базовые стили для мобильного, затем расширяй под большие экраны.

```css
.container {
  padding: 16px;
}

@media (min-width: 768px) {
  .container {
    padding: 24px;
  }
}
```

Так проще поддерживать и меньше конфликтов.

---

## Units: что когда

- `px` — точные значения для границ, иконок, мелких деталей.
- `rem` — размеры шрифта, отступы и масштабируемые размеры интерфейса.
- `%` — относительные ширины/высоты внутри контейнера.
- `vw/vh` — привязка к viewport.
- `dvh/svh/lvh` — более предсказуемая высота на мобильных.

Практика:
- типографика и spacing чаще в `rem`;
- ширины компонентов часто `%`/`fr`/`minmax()`;
- большие герои и full-screen блоки аккуратно через `dvh`.

---

## `clamp()` для гибких значений

```css
.hero__title {
  font-size: clamp(1.75rem, 1.2rem + 2vw, 3rem);
}
```

`clamp(min, preferred, max)` позволяет не плодить лишние брейкпоинты.

---

## Брейкпоинты

Не копируй "магические" числа из чужих проектов.

Опирайся на:
- поведение контента;
- переломы layout;
- реальные устройства аудитории.

Чаще достаточно 2-4 ключевых точек.

---

## Что проверять в DevTools

- 320 / 375 / 768 / 1024 / 1440;
- горизонтальный скролл;
- длину заголовков и кнопок на реальном контенте;
- состояние меню, форм, таблиц, карточек.

---

## Частые ошибки

- Сначала верстают desktop, потом "ломают" всё на mobile.
- Везде используют `px`, из-за чего интерфейс плохо масштабируется.
- Дублируют медиазапросы без системы.
- Не проверяют промежуточные ширины между брейкпоинтами.

---

## 🔧 Практика

1. Сделай mobile-first страницу с 3 брейкпоинтами.
2. Примени `clamp()` к заголовку и вертикальным отступам.
3. Прогони проверку на 5 ширинах и составь список багов.
