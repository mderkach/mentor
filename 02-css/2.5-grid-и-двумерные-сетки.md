# 2.5 Grid и двумерные сетки

Grid — лучший инструмент, когда нужно управлять layout сразу по двум осям: строкам и колонкам.

---

## 🎯 Что ты освоишь

- как задавать колонки/строки и gaps;
- как использовать `fr`, `minmax()`, `repeat()`;
- когда брать `grid`, а когда `flex`;
- как делать адаптивные сетки с `auto-fit`/`auto-fill`.

---

## База Grid

```css
.cards {
  display: grid;
  grid-template-columns: repeat(3, minmax(0, 1fr));
  gap: 16px;
}
```

Ключевые идеи:
- `fr` делит свободное пространство;
- `minmax()` ограничивает поведение трека;
- `gap` управляет отступами между ячейками.

---

## `auto-fit` и `auto-fill` простыми словами

```css
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
  gap: 16px;
}
```

- `auto-fit` старается "схлопнуть" пустые колонки;
- `auto-fill` резервирует колонки даже без контента.

Для карточных сеток чаще подходит `auto-fit`.

---

## Grid areas для крупных блоков

```css
.layout {
  display: grid;
  grid-template-columns: 260px 1fr;
  grid-template-areas:
    "sidebar content";
  gap: 24px;
}

.layout__sidebar {
  grid-area: sidebar;
}

.layout__content {
  grid-area: content;
}
```

Так структура страницы читается как схема, а не как набор случайных чисел.

---

## Flex vs Grid

- `flex` -> одномерные задачи (ряд/колонка).
- `grid` -> двумерные layout (и строки, и колонки важны).

Если строишь каталог, дашборд или сложную страницу с зонами, чаще нужен `grid`.

---

## Частые ошибки

- Пытаются имитировать grid через вложенные flex-контейнеры.
- Ставят фиксированные ширины колонок без `minmax()`.
- Забывают, что контент может быть длиннее макета.
- Не проверяют сетку на мобильных ширинах.

---

## 🔧 Практика

1. Собери сетку карточек с авто-переносом через `auto-fit`.
2. Сделай layout "sidebar + content" через `grid-template-areas`.
3. Переключи сетку на 1 колонку на мобильном и проверь читабельность.
