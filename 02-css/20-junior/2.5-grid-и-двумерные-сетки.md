# 2.5 Grid и двумерные сетки

Grid — лучший инструмент, когда нужно управлять layout сразу по двум осям: строкам и колонкам.

---

## Что уже должно быть знакомо

- одномерный layout через [2.4 Flexbox](../10-intern/2.4-flexbox.md);
- базовая геометрия, overflow и размеры из [2.3 Поток документа, блочная модель и позиционирование](../10-intern/2.3-поток-документа-блочная-модель-и-позиционирование.md).

---

## 🎯 Что ты освоишь

- как задавать колонки/строки и gaps;
- как использовать `fr`, `minmax()`, `repeat()`;
- когда брать `grid`, а когда `flex`;
- как делать адаптивные сетки с `auto-fit`/`auto-fill`;
- как применять `grid-auto-flow`, `grid-column/grid-row` и `subgrid`.

---

## Что важно понять

- Grid нужен не для «раскладывания по пикселям», а для устойчивого двухмерного layout-контракта.
- Хорошая сетка должна переживать реальный контент, а не только идеальный макет.
- Если flex отвечает за выравнивание элементов в одном направлении, grid отвечает за структуру страницы или блока целиком.

---

## База Grid

```css
.cards {
  display: grid;
  grid-template-columns: repeat(3, minmax(0, 1fr));
  gap: 16px;
}
```

Ключевые идеи:

- `fr` делит свободное пространство;
- `minmax()` ограничивает поведение трека;
- `gap` управляет отступами между ячейками.

Не забывай про строки:

```css
.dashboard {
  display: grid;
  grid-template-columns: 240px 1fr;
  grid-template-rows: auto 1fr;
}
```

---

## `auto-fit` и `auto-fill` простыми словами

```css
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
  gap: 16px;
}
```

- `auto-fit` старается "схлопнуть" пустые колонки;
- `auto-fill` резервирует колонки даже без контента.

Для карточных сеток чаще подходит `auto-fit`.

---

## Grid areas для крупных блоков

```css
.layout {
  display: grid;
  grid-template-columns: 260px 1fr;
  grid-template-areas:
    "sidebar content";
  gap: 24px;
}

.layout__sidebar {
  grid-area: sidebar;
}

.layout__content {
  grid-area: content;
}
```

Так структура страницы читается как схема, а не как набор случайных чисел.

---

## Явное размещение и авто-поток

```css
.grid {
  display: grid;
  grid-template-columns: repeat(4, minmax(0, 1fr));
  justify-items: stretch;
  grid-auto-flow: row dense;
}

.grid__hero {
  grid-column: 1 / -1;
  justify-self: start;
}
```

- `grid-column`/`grid-row` задают позицию элемента по линиям;
- `justify-items` выравнивает все элементы по inline-оси внутри ячеек;
- `justify-self` переопределяет выравнивание для конкретного элемента;
- `grid-auto-flow` управляет автозаполнением и может плотнее упаковать карточки.

---

## `subgrid`: когда сетка внутри должна подчиняться родителю

`subgrid` полезен, когда дочерние карточки должны выровняться по тем же колонкам, что и родительская сетка.

```css
.catalog {
  display: grid;
  grid-template-columns: repeat(12, 1fr);
  gap: 16px;
}

.product-card {
  display: grid;
  grid-template-columns: subgrid;
  grid-column: span 4;
}
```

Если `subgrid` недоступен для части целевых браузеров, делай fallback через обычные `grid-template-columns`.

---

## Flex vs Grid

- `flex` -> одномерные задачи (ряд/колонка).
- `grid` -> двумерные layout (и строки, и колонки важны).

Если строишь каталог, дашборд или сложную страницу с зонами, чаще нужен `grid`.

---

## Частые ошибки

- Пытаются имитировать grid через вложенные flex-контейнеры.
- Ставят фиксированные ширины колонок без `minmax()`.
- Забывают, что контент может быть длиннее макета.
- Не проверяют сетку на мобильных ширинах.

---

## Anti-pattern

"Сетка совпала с макетом на одном экране, значит layout готов".

Для production этого мало: grid нужно проверять на длинном контенте, узких ширинах и реальном порядке блоков.

---

## 🔧 Практика

1. Собери сетку карточек с авто-переносом через `auto-fit`.
1. Сделай layout "sidebar + content" через `grid-template-areas`.
1. Добавь `subgrid` или fallback для выравнивания внутренних карточек.
1. Переключи сетку на 1 колонку на мобильном и проверь читабельность.
