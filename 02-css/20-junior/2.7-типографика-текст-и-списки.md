# 2.7 Типографика, текст и списки

Текст — главный интерфейс продукта. Плохая типографика ломает UX даже при идеальном layout.

---

## Что уже должно быть знакомо

- базовая семантика текста и списков в HTML: [1.1.3 Семантика, скринридер и встроенные HTML API](../../01-html/10-intern/1.1.3-Семантика-скринридер-и-встроенные-API.md);
- размеры, box model и overflow из [2.3 Поток документа, блочная модель и позиционирование](../10-intern/2.3-поток-документа-блочная-модель-и-позиционирование.md).

---

## 🎯 Что ты освоишь

- как собирать читабельную типографику;
- как управлять межстрочностью, длиной строки и акцентами;
- как стилизовать списки без потери смысла;
- как избежать типографических багов на адаптиве;
- как использовать `accent-color`, `color-scheme`, `line-clamp`.

---

## Что важно понять

- Типографика в продукте — это не «оформление текста», а часть навигации, иерархии и скорости чтения.
- Если текст тяжело читать, интерфейс уже плохой, даже если layout технически корректен.
- Работа со списками и заголовками должна сохранять смысловую структуру, а не маскировать ее декоративными приемами.

---

## Базовые правила читаемости

- Размер основного текста обычно `16px` (`1rem`) и выше.
- Комфортная `line-height` для абзацев: `1.4-1.7`.
- Оптимальная длина строки: примерно 45-80 символов.

```css
.article {
  font-size: 1rem;
  line-height: 1.6;
  max-width: 70ch;
}
```

Цвета и размеры лучше сразу привязывать к токенам, чтобы потом не мигрировать вручную. Больше о токенах: [2.12 CSS Custom Properties и Design Tokens](../30-middle/2.12-CSS-custom-properties-и-design-tokens.md).

---

## Тема браузера и элементы форм

```css
:root {
  color-scheme: light dark;
}

input[type='checkbox'] {
  accent-color: #1a73e8;
}

input,
textarea {
  caret-color: #1a73e8;
}
```

- `color-scheme` помогает нативным элементам подстроиться под светлую/тёмную тему;
- `accent-color` задаёт системный акцент (checkbox/radio/progress);
- `caret-color` повышает читаемость курсора в полях.

Это уже не только про красоту темы, но и про уважение системных настроек пользователя.
Глубже связка `color-scheme`, `accent-color`, high contrast и пользовательских preferences разобрана в [2.14 CSS accessibility, system preferences и адаптивное взаимодействие](../30-middle/2.14-CSS-accessibility-system-preferences-и-адаптивное-взаимодействие.md).

## Подключение шрифтов: `@font-face`

```css
@font-face {
  font-family: 'InterVar';
  src: url('/fonts/inter-var.woff2') format('woff2');
  font-weight: 100 900;
  font-display: swap;
}
```

- `font-display: swap` снижает риск "невидимого текста";
- подключай только реально используемые начертания, иначе растёт вес страницы.

---

## Иерархия заголовков

UI должен явно отвечать на вопрос "что здесь главное".

Практика:

- 2-3 уровня заголовков в одном блоке обычно достаточно;
- разница между уровнями должна быть заметной;
- не полагайся только на цвет, используй размер/жирность/интервалы.

---

## Текстовые свойства, которые реально нужны

- `font-size`, `font-weight`, `line-height`;
- `letter-spacing` (очень аккуратно);
- `text-wrap: balance` для заголовков, где поддерживается;
- `line-clamp` для аккуратной многострочной обрезки карточек;
- `text-overflow: ellipsis` для обрезки однострочного текста;
- `overflow-wrap` и `word-break` для длинных слов/ссылок;
- `hyphens: auto` для длинных слов (если язык задан).

```css
.news-card__title {
  display: -webkit-box;
  -webkit-line-clamp: 2;
  -webkit-box-orient: vertical;
  overflow: hidden;
}
```

---

## Списки

Списки — это структура, а не просто "точки".

```css
.features {
  list-style-type: disc;
  list-style-position: outside;
  padding-left: 1.25rem;
}

.features li::marker {
  color: #1a73e8;
}
```

Не убирай маркеры просто "потому что мешают" — сначала проверь, не теряется ли смысл.

---

## Частые ошибки

- Слишком мелкий текст в карточках/формах.
- Узкая межстрочность, из-за которой тяжело читать.
- Заголовки и обычный текст почти не отличаются.
- Стилизация списков через `div`, а не через `ul/ol`.

---

## Anti-pattern

"Текстовый блок выглядит скучно, значит нужно сильнее украсить его стилями".

Обычно продуктовой типографике важнее читаемость и иерархия, чем декоративность.

---

## 🔧 Практика

1. Настрой типографическую шкалу для страницы: `h1/h2/h3/body/small`.
1. Оформи статью с ограничением строки через `ch`.
1. Сделай список преимуществ с аккуратными маркерами через `::marker`.

---
