# 2.2 Селекторы, псевдоклассы и псевдоэлементы

Селекторы определяют, **кому** применять стиль. Чем аккуратнее они построены, тем стабильнее кодовая база.

---

## Что уже должно быть знакомо

- база каскада, наследования и специфичности из [2.1 Базовые знания CSS](./2.1-базовые-знания-css.md).

---

## 🎯 Что ты освоишь

- как выбирать селекторы без перегруза специфичности;
- где использовать `:hover`, `:focus`, `:focus-visible`, `:disabled`;
- как применять `::before` и `::after` без ломания доступности;
- как читать структурные селекторы (`:nth-*`, `:first-*`, `:last-*`);
- как читать селектор и сразу понимать риск.

---

## Что важно понять

- Селектор задаёт не только внешний вид, но и границу влияния правила.
- Плохой селектор ломает масштабирование CSS быстрее, чем отсутствие очередной «удобной» фичи.
- Псевдоклассы и псевдоэлементы нужны не для декоративной магии, а для управляемых состояний и аккуратного HTML.

---

## База селекторов

Чаще всего хватает:

- `.class`;
- `element` (осторожно);
- `[attr]` и `[attr="value"]`;
- сочетаний `.block .element` при внятной структуре.

Обычно лучше опираться на классы, а не на глубокую вложенность HTML.

---

## Псевдоклассы состояния

Ключевые для UI:

- `:hover` — наведение;
- `:focus` / `:focus-visible` — фокус клавиатуры;
- `:active` — активное нажатие;
- `:disabled` — недоступный контрол.

Пример:

```css
.button {
  background: #1a73e8;
  color: #fff;
}

.button:hover {
  background: #1557b0;
}

.button:focus-visible {
  outline: 2px solid #0b57d0;
  outline-offset: 2px;
}

.button:disabled {
  opacity: 0.5;
  cursor: not-allowed;
}
```

`focus-visible` важно не только "чтобы outline был".
Это часть доступности интерфейса: клавиатурная навигация, разные способы ввода и системные пользовательские настройки дальше отдельно собраны в [2.14 CSS accessibility, system preferences и адаптивное взаимодействие](../30-middle/2.14-CSS-accessibility-system-preferences-и-адаптивное-взаимодействие.md).

Для форм также часто нужны:

- `:checked`;
- `:required` / `:optional`;
- `:valid` / `:invalid`;
- `:placeholder-shown`;
- `:focus-within` (состояние контейнера поля).
- `:target` / `:target-within` (якорная навигация и deep-link в интерфейсе).

---

## Структурные селекторы без боли

Частый минимум:

- `:first-child` / `:last-child`;
- `:nth-child()` / `:nth-of-type()`;
- `:only-child`;
- `:empty`.

```css
.menu__item:not(:last-child) {
  margin-right: 12px;
}

.table__row:nth-child(even) {
  background: #f8fafc;
}
```

Главное правило: если селектор сложно прочитать вслух, лучше добавить класс.

---

## Псевдоэлементы

- `::before` и `::after` — декоративные элементы;
- `::marker` — маркер списков;
- `::placeholder` — плейсхолдер формы;
- `::selection` — стиль выделения текста.

Для более продвинутых сценариев (обычно middle+):

- `::first-letter`, `::first-line`;
- `::file-selector-button` для `input[type="file"]`;
- `::backdrop` для диалогов;
- `::target-text` для стилизации найденного фрагмента текста;
- `::part()` и `::slotted()` в Web Components.

Важно:

- не клади в `::before/::after` важный смысловой текст;
- декоративный контент должен оставаться декоративным.

Если нужна база по тому, для кого вообще важна доступность и почему "не ломать клавиатуру" не сводится к одному screen reader, смотри: [1.1.3 Семантика, скринридер и встроенные HTML API](../../01-html/10-intern/1.1.3-Семантика-скринридер-и-встроенные-API.md#для-кого-вообще-нужна-доступность).

---

## `:is()`, `:where()`, `:not()`

- `:is()` — сокращает повторяющиеся селекторы;
- `:where()` — как `:is()`, но с нулевой специфичностью;
- `:not()` — исключает случаи.

Практика для масштабируемого CSS:

- сложные наборы селекторов выноси в `:is()`;
- низкую специфичность базовых правил держи через `:where()`.
- `:has()` внедряй через progressive enhancement и проверку поддержки. Больше о modern CSS-подходе: [2.10 Современный CSS 2024-2026](../30-middle/2.10-современный-css-2024-2026.md).

---

## Частые ошибки

- Используют длинные цепочки селекторов вместо понятного класса.
- Ставят стили только на `:hover`, забывая клавиатурный `:focus-visible`.
- Через `::before` вставляют текст, который реально нужен пользователю.
- Лечат конфликт специфичности добавлением ещё более тяжёлого селектора.

---

## Anti-pattern

"Чем длиннее селектор, тем надёжнее правило".

На практике длинные селекторы чаще делают CSS хрупким и усложняют переопределение состояний.

---

## 🔧 Практика

1. Перепиши 5 длинных селекторов в более короткие и читаемые.
1. Добавь корректные состояния `hover/focus-visible/disabled` для кнопки и ссылки.
1. Оформи список через `::marker` без лишней вложенности в HTML.

---

## ✅ Чеклист

- [ ] Умею выбирать селектор без лишней специфичности
- [ ] Для интерактива покрыты `:hover` и `:focus-visible`
- [ ] Понимаю разницу между псевдоклассом и псевдоэлементом
- [ ] Использую `:where()` и `:is()` осознанно
