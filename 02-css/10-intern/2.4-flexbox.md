# 2.4 Flexbox

Flexbox — основной инструмент для одномерных layout: ряд или колонка, выравнивание, распределение пространства.

---

## Что уже должно быть знакомо

- поток, размеры и позиционирование из [2.3 Поток документа, блочная модель и позиционирование](./2.3-поток-документа-блочная-модель-и-позиционирование.md).

---

## 🎯 Что ты освоишь

- как работает ось `main` и `cross`;
- как выравнивать элементы без костылей;
- где `flex` идеален, а где лучше `grid`;
- почему `min-width: 0` иногда спасает интерфейс;
- как использовать `align-content`, `place-*`, `row-gap/column-gap`.

---

## Что важно понять

- Flexbox решает одномерные задачи: либо ряд, либо колонка как основная ось.
- Выравнивание в flex всегда начинается с понимания `main axis` и `cross axis`, а не с перебора свойств наугад.
- Большинство flex-багов возникают не из-за самой модели, а из-за непонимания, что задается контейнеру, а что элементу.

---

## База: контейнер и элементы

```css
.list {
  display: flex;
  gap: 12px;
  align-items: center;
}
```

Ключевые свойства контейнера:

- `flex-direction`;
- `justify-content`;
- `align-items`;
- `flex-wrap`;
- `flex-flow`;
- `gap`.

Ключевые свойства элемента:

- `flex-grow`;
- `flex-shrink`;
- `flex-basis`;
- `align-self`.
- `place-items`, `place-content`, `place-self`.

---

## Выравнивание в несколько строк

Когда есть перенос (`flex-wrap: wrap`), подключается `align-content`:

```css
.chips {
  display: flex;
  flex-flow: row wrap;
  row-gap: 8px;
  column-gap: 12px;
  align-content: start;
}
```

Если контейнер без переноса, `align-content` не даст эффекта.

---

## Когда использовать flex

Лучшие кейсы:

- navbar;
- строка кнопок;
- карточка "контент + действия";
- вертикальное выравнивание в небольших компонентах.

Когда лучше не flex:

- сложная двумерная сетка с контролем и по строкам, и по колонкам -> это `grid`.

Больше о двумерных сетках и `subgrid`: [2.5 Grid и двумерные сетки](../20-junior/2.5-grid-и-двумерные-сетки.md).

---

## Частый баг: текст не сжимается

Если внутри flex-элемента длинный текст, он может выталкивать layout.

Решение:

```css
.item {
  min-width: 0;
}

.item__title {
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
}
```

---

## Паттерн "липкий футер"

```css
.page {
  min-height: 100dvh;
  display: flex;
  flex-direction: column;
}

.page__main {
  flex: 1;
}
```

Этот подход надёжнее, чем хаки с абсолютным позиционированием футера.

---

## Частые ошибки

- Путают `justify-content` и `align-items`.
- Используют `margin` вместо `gap` для всех случаев.
- Не включают `wrap`, когда элементы должны переноситься.
- Пытаются строить полноценную сетку через вложенные flex-контейнеры.

---

## Anti-pattern

"Flexbox подходит для любого layout, если достаточно вложить контейнеров".

Нет. Как только тебе важны и строки, и колонки, почти всегда пора переходить к grid.

---

## 🔧 Практика

1. Собери header: логотип слева, меню по центру, кнопка справа.
1. Сделай ряд карточек с переносом на новую строку через `flex-wrap`.
1. Исправь layout с длинным заголовком через `min-width: 0`.
