# 2.1 Базовые знания CSS

CSS отвечает не только за "красиво", но и за предсказуемость интерфейса: почему стиль применился, почему не применился и почему всё внезапно поехало.

## Что уже должно быть знакомо

- Базовая работа с DevTools из [0.0.1 CSS в DevTools: каскад, слои и layout-debug](../00-zero/0.0.1-CSS-в-DevTools-каскад-слои-layout-debug.md).

---

## 🎯 Что ты освоишь

- как браузер выбирает итоговый стиль (каскад);
- чем отличаются наследование, специфичность и порядок в файле;
- что делают `initial`, `inherit`, `unset`, `revert`;
- зачем нужен `box-sizing: border-box`;
- как не воевать с `!important`.

---

## Термины урока

- `Каскад` — это алгоритм, по которому браузер выбирает итоговое CSS-правило среди конкурирующих.
- `Специфичность` — это «вес» селектора, который влияет на победу в каскаде.
- `Наследование` — это механизм, когда часть свойств (обычно текстовые) переходит от родителя к потомку.

---

## Что важно понять

- CSS в базе надо воспринимать не как список свойств, а как систему выбора победившего правила.
- Большая часть ранних CSS-багов возникает не из-за «странного браузера», а из-за непонимания каскада, наследования и порядка источников.
- Если база здесь не собрана, все следующие уроки про layout, grid и modern CSS будут ощущаться как набор трюков.

---

## Как браузер применяет стили на самом деле

Когда несколько правил подходят к одному элементу, браузер проходит по шагам:

1. Важность (`!important`/обычное правило).
1. Источник (стили браузера, пользовательские, авторские).
1. Специфичность селектора.
1. Порядок объявления (кто ниже, тот победил).

Это и есть каскад. Если его понимать, 80% CSS-багов чинятся быстро.

---

## Наследование: что берётся от родителя

Наследуются в основном текстовые свойства:

- `color`;
- `font-family`;
- `font-size` (если не переопределить).

Не наследуются обычно layout-свойства:

- `margin`, `padding`, `border`, `width`, `display`.

Практический вывод:

- цвет и шрифт удобно задавать выше по дереву;
- размеры и отступы почти всегда задаём конкретному блоку.

---

## Специфичность без боли

- `#id` сильнее `.class`;
- `.class` сильнее `tag`;
- универсальный `*` почти ничего не весит.

Что делать в реальном проекте:

- держи селекторы короткими;
- не строй "цепочки мести" вроде `.page .content .card .title`;
- избегай `!important`, пока есть нормальное решение через архитектуру селекторов.

---

## Ключевые слова сброса и наследования

- `initial` — сбрасывает свойство к стандартному значению из спецификации.
- `inherit` — принудительно наследует значение от родителя.
- `unset` — как "умный сброс": наследует наследуемые свойства и сбрасывает остальные.
- `revert` — откатывает значение к предыдущему источнику каскада (например, к user-agent стилям).
- `revert-layer` — откатывает значение к предыдущему каскадному слою.

```css
.button {
  color: inherit;
  border: unset;
}

.legacy-widget * {
  all: revert;
}
```

`revert-layer` полезен там, где уже используешь `@layer`. Больше о слоях каскада: [2.10 Современный CSS 2024-2026](../30-middle/2.10-современный-css-2024-2026.md).

---

## Базовый reset для старта

```css
* {
  box-sizing: border-box;
}

*::before,
*::after {
  box-sizing: inherit;
}

body {
  margin: 0;
  font-family: system-ui, sans-serif;
}
```

Почему это важно:

- `border-box` делает расчёт размеров предсказуемым;
- отключение дефолтных отступов `body` убирает неожиданные "белые поля".

---

## Когда допустим `!important`

Нормально использовать точечно:

- утилитарные классы (`.sr-only`, `.hidden`);
- форс-мажорное переопределение внешней библиотеки;
- состояние, которое обязательно должно победить.

Плохая практика:

- лечить `!important` каждый конфликт каскада.

---

## Частые ошибки новичков

- Пытаются "угадать" стиль вместо чтения DevTools.
- Используют `!important` как основной инструмент.
- Путают наследование и специфичность.
- Не понимают, что "правило ниже" выигрывает при равной специфичности.

---

## Anti-pattern

"Если стиль не применился, значит надо сделать селектор сильнее".

Обычно нет. Сначала нужно понять, кто выигрывает в каскаде и почему.

---

## 🔧 Практика

1. Создай 3 конкурирующих правила для одного элемента и объясни, какое победило и почему.
1. Сделай мини-страницу и задай шрифт/цвет через `body`, а отступы через конкретные блоки.
1. Найди баг с `!important` и перепиши без него.
