# 2.7 Типографика, текст и списки

Текст — главный интерфейс продукта. Плохая типографика ломает UX даже при идеальном layout.

---

## 🎯 Что ты освоишь

- как собирать читабельную типографику;
- как управлять межстрочностью, длиной строки и акцентами;
- как стилизовать списки без потери смысла;
- как избежать типографических багов на адаптиве.

---

## Базовые правила читаемости

- Размер основного текста обычно `16px` (`1rem`) и выше.
- Комфортная `line-height` для абзацев: `1.4-1.7`.
- Оптимальная длина строки: примерно 45-80 символов.

```css
.article {
  font-size: 1rem;
  line-height: 1.6;
  max-width: 70ch;
}
```

---

## Иерархия заголовков

UI должен явно отвечать на вопрос "что здесь главное".

Практика:
- 2-3 уровня заголовков в одном блоке обычно достаточно;
- разница между уровнями должна быть заметной;
- не полагайся только на цвет, используй размер/жирность/интервалы.

---

## Текстовые свойства, которые реально нужны

- `font-size`, `font-weight`, `line-height`;
- `letter-spacing` (очень аккуратно);
- `text-wrap: balance` для заголовков, где поддерживается;
- `text-overflow: ellipsis` для обрезки однострочного текста;
- `hyphens: auto` для длинных слов (если язык задан).

---

## Списки

Списки — это структура, а не просто "точки".

```css
.features {
  padding-left: 1.25rem;
}

.features li::marker {
  color: #1a73e8;
}
```

Не убирай маркеры просто "потому что мешают" — сначала проверь, не теряется ли смысл.

---

## Частые ошибки

- Слишком мелкий текст в карточках/формах.
- Узкая межстрочность, из-за которой тяжело читать.
- Заголовки и обычный текст почти не отличаются.
- Стилизация списков через `div`, а не через `ul/ol`.

---

## 🔧 Практика

1. Настрой типографическую шкалу для страницы: `h1/h2/h3/body/small`.
2. Оформи статью с ограничением строки через `ch`.
3. Сделай список преимуществ с аккуратными маркерами через `::marker`.

---

## ✅ Чеклист

- [ ] Текст читается комфортно на desktop и mobile
- [ ] Есть явная визуальная иерархия заголовков
- [ ] Длина строки контролируется
- [ ] Списки остаются семантичными
