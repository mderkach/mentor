# Заметка на будущее: инструменты фронтенд-разработчика

Этот файл не входит в текущий HTML-трек. Это отдельная памятка, чтобы позже развить тему окружения.

---

## Зачем это понадобится

После базовой HTML/CSS-вёрстки джуну нужно рабочее окружение:

- единое форматирование;
- проверка качества кода;
- предсказуемая работа с Git и сборкой;
- одинаковые правила для всей команды.

---

## Базовый стек (черновик)

- IDE: VS Code
- Runtime/пакеты: Node.js
- Контроль версий: Git
- Форматирование: Prettier
- Линтинг JS/TS: ESLint
- Линтинг стилей: Stylelint
- Единые настройки редакторов: EditorConfig

---

## Что добавить в будущий урок

1. Установка и базовая настройка расширений VS Code.
2. Минимальный набор конфигов (`.editorconfig`, `.prettierrc`, `eslint`, `stylelint`).
3. Скрипты `lint`, `format`, `lint:fix`.
4. Проверка качества перед PR.
5. Типичные ошибки джуна в окружении и как их быстро чинить.
