# 1.3.3 Семантика больших страниц и информационная архитектура

На мидл-уровне важно не просто ставить правильные теги, а проектировать целостную структуру страницы так, чтобы её было удобно развивать годами.

---

## 🎯 Что ты освоишь

- декомпозицию длинных страниц на смысловые зоны;
- правила построения heading-иерархии для сложного контента;
- устойчивую разметку для повторного использования блоков.

---

## Подход

1. Сначала карта контента (какие зоны и зачем).
2. Потом ландмарки (`header/main/aside/footer/nav`).
3. Затем иерархия заголовков.
4. И только после этого — стили и интерактив.

---

## Пример мысленного фреймворка

- Hero
- Value proposition
- Feature sections
- FAQ
- CTA

Каждый блок имеет свой семантический контейнер и уровень заголовка.

---

## Ошибки

- HTML формируется «по пикселям макета», а не по смыслу.
- Заголовки прыгают хаотично.
- Повторяющиеся блоки имеют разную структуру в разных местах.

---

## 🔧 Практика

1. Возьми длинный лендинг и сделай его semantic map.
2. Приведи заголовки к строгой иерархии.
3. Сформируй единый шаблон для повторяющихся секций.

---

## ✅ Чеклист

- [ ] Есть semantic map страницы
- [ ] Heading-структура предсказуема
- [ ] Повторяемые блоки стандартизированы
