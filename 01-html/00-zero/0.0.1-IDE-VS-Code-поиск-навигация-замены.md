# 0.0.1 VS Code как IDE: поиск, навигация, массовые замены

Этот урок 0 нужен, чтобы ты сразу работал как инженер, а не как «человек-блокнот».

Блокнот умеет только редактировать текст.
IDE (VS Code) помогает управлять кодовой базой: быстро находить, менять, проверять и не ломать проект.

---

## 🎯 Что ты освоишь

- зачем IDE лучше блокнота в реальной разработке;
- как искать по файлу, по проекту, по строке, по имени файла;
- как делать массовые замены безопасно;
- как использовать regex в поиске и замене на практике.

---

## Термины урока

- `IDE` — это среда разработки, где ты пишешь, ищешь, проверяешь и рефакторишь код.
- `Regex` — это шаблон поиска, который позволяет находить и заменять группы строк по правилам.
- `Массовая замена` — это изменение множества совпадений в проекте за один проход с предпросмотром diff.

---

## Почему IDE лучше блокнота

Что даёт VS Code, чего обычно нет в блокноте:

- навигация по проекту и файлам;
- умный поиск по коду;
- массовые замены с предпросмотром;
- подсветка синтаксиса и ошибок;
- быстрая работа с Git, терминалом и расширениями.

На больших задачах это экономит часы, а не минуты.

## Быстрая навигация (must-have)

### Открыть файл по имени

- macOS: `Cmd + P`
- Windows/Linux: `Ctrl + P`

Пиши часть имени файла, например: `header`, `form`, `README`.

### Переход к строке

- macOS/Windows/Linux: `Ctrl/Cmd + G`

Удобно, когда в ревью или логе указан номер строки.

### Переход к символу в файле

- macOS: `Cmd + Shift + O`
- Windows/Linux: `Ctrl + Shift + O`

Быстрый переход к заголовкам, функциям и структуре текущего файла.

---

## Поиск и замена в одном файле

### Поиск

- macOS: `Cmd + F`
- Windows/Linux: `Ctrl + F`

### Замена

- macOS: `Option + Cmd + F`
- Windows/Linux: `Ctrl + H`

Кейс:

- нужно заменить `card-title` на `product-card__title` в одном файле.

---

## Поиск и замена по всему проекту

### Поиск по проекту

- macOS: `Cmd + Shift + F`
- Windows/Linux: `Ctrl + Shift + F`

### Замена по проекту

- macOS: `Cmd + Shift + H`
- Windows/Linux: `Ctrl + Shift + H`

Главное правило:

- всегда смотри **предпросмотр изменений** перед `Replace All`.

## Regex и массовые замены: объяснение для новичка

Включи режим регулярных выражений (кнопка `.*` в поиске).

Если коротко:

- `()` — «запомни кусок текста»;
- `$1`, `$2` — вставь запомненный кусок в замену;
- `.` — любой символ;
- `*` — повторяется 0 и более раз;
- `\` — экранирование (когда нужно искать спецсимвол буквально).

### Пример 1 (подробно). Переименовать класс во всех файлах

Ситуация:

- было: `class="card-title"`
- нужно: `class="product-card__title"`

Как сделать:

1. Открой поиск по проекту: `Cmd/Ctrl + Shift + F`.
1. В поле поиска вставь: `class="card-title"`
1. В поле замены вставь: `class="product-card__title"`
1. Нажми `Replace All` только после предпросмотра изменений.

Это самый понятный и безопасный сценарий массовой замены для старта.

### Пример 1b (чуть сложнее). Когда у элемента несколько классов

Ситуация:

- было: `class="card card-title mb-16"`
- нужно: заменить только `card-title`, не трогая остальные классы.

Поиск:

```regex
class="([^"]*\b)card-title(\b[^"]*)"
```

Замена:

```text
class="$1product-card__title$2"
```

Что происходит:

- `([^"]*\b)` — запоминаем всё, что было до `card-title`;
- `(\b[^"]*)` — запоминаем всё, что было после `card-title`;
- в замене собираем строку обратно и меняем только нужный фрагмент.

### Пример 2. Переименовать BEM-модификатор и сохранить суффикс

Поиск:

```regex
btn--([a-z-]+)
```

Замена:

```text
button--$1
```

Работает так:

- `btn--primary` -> `button--primary`
- `btn--danger` -> `button--danger`

### Пример 3. Закомментировать все TODO-строки

Поиск:

```regex
^(\s*)(TODO:.*)$
```

Замена:

```text
$1<!-- $2 -->
```

### Пример 4. Найти все `console.log` перед ревью

Поиск:

```regex
console\.log\([^)]*\)
```

Сначала просто найди, потом реши: удалить или заменить на нормальный логгер.

## Безопасные массовые замены

1. Сначала ограничь область поиска (папка/маска файлов).
1. Сделай маленькую пробную замену.
1. Проверь diff.
1. Только потом `Replace All`.

Если сомневаешься — делай замену частями.
