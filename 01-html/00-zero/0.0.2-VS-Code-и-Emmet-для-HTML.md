# 0.0.2 VS Code и Emmet для HTML

Если ты пишешь HTML вручную символ за символом, ты тратишь время не на обучение, а на механическую работу.

Этот урок нужен, чтобы ты уже с первого дня писал разметку быстро и без боли.

---

## 🎯 Что ты освоишь

- как включить Emmet в VS Code так, чтобы он реально ускорял работу;
- 10 аббревиатур, которые покрывают 80% повседневной вёрстки;
- как быстро оборачивать существующий код в нужные теги;
- как не ломать структуру страницы, когда генерируешь разметку автоматически.

---

## Зачем это важно

Представь: тебе дали макет с 30 карточками, списками, кнопками, блоками формы. Это очень долго все писать вручную и не продуктивно.

Чтобы решить это проблему, был создан Emmet — это мощный плагин для текстовых редакторов (VS Code, Sublime Text, IDE от JetBrains и др.), предназначенный для кардинального ускорения написания HTML, CSS, XML и XSL кода. Он позволяет преобразовывать короткие аббревиатуры в полноценные блоки кода при нажатии клавиши Tab, значительно повышая продуктивность верстки

Без Emmet ты печатаешь каждую пару тегов вручную.
С Emmet ты описываешь структуру одной строкой и сразу получаешь каркас.

Это экономит десятки минут в день и снижает количество опечаток.

Раньше Emmet приходилось ставить вручную (сейчас только какие-либо экслюзивных редакторов), но сейчас - это стандарт индустрии и уже предустановлен вместе с IDE

---

## Шаг 1. Настрой VS Code

> ⚠️ В целом данный функционал включен по умолчанию, но если вдруг не работает - сделай эти настройки

Открой `settings.json` и добавь:

```json
{ 
  "emmet.triggerExpansionOnTab": true,
  "emmet.showSuggestionsAsSnippets": true,
  "editor.snippetSuggestions": "top"
}
```

Что это даст:

- `Tab` разворачивает аббревиатуры;
- подсказки Emmet поднимаются выше;
- структура генерируется быстрее, чем ты успеваешь печатать руками.

---

## Шаг 2. Минимальный набор Emmet

### 1) `!`

```emmet
!
```

Разворачивается в [базовый HTML-каркас](../10-intern/1.1.1-Каркас-документа-и-структура-страницы.md) (`doctype`, `html`, `head`, `body`).

### 2) Вложенность через `>`

```emmet
header>nav>ul>li*3>a
```

### 3) Повторение через `*`

```emmet
.card*4
```

### 4) Нумерация через `$`

```emmet
li.item$*5
```

### 5) Текст через `{}`

```emmet
a{Подробнее}
```

### 6) Класс через `.` и id через `#`

```emmet
section.hero#main
```

### 7) Братья через `+`

```emmet
h2+p+button
```

### 8) Поднятие уровня через `^`

```emmet
ul>li*3^p
```

### 9) Группировка через `()`

```emmet
main>(section>h2+p)*2
```

### 10) Атрибуты через `[]`

```emmet
input[type="email" name="email" required]
```

## Шаг 3. Wrap with Abbreviation

Это must-have при рефакторинге.

Сценарий:

1. Выделяешь существующий кусок HTML.
1. Запускаешь команду `Emmet: Wrap with Abbreviation`.
1. Вводишь, например: `section.card>div.card__body`.

И код моментально получает новую структуру без ручной перепечатки.

---

## Типичные ошибки новичков

- Нажимают `Tab`, но не включили `emmet.triggerExpansionOnTab`.
- Генерируют большую структуру, не понимая, что она означает семантически.
- Забывают проверять результат: Emmet ускоряет, но не думает за тебя.

---

## 🔧 Практика

1. Сгенерируй каркас страницы через `!`.
1. Сверстай шапку через `header>nav>ul>li*4>a{Пункт $}`.
1. Сверстай сетку карточек через `main>section.cards>article.card*6`.
1. Оберни блок контента в `section.content` через Wrap with Abbreviation.
