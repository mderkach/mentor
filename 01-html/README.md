# 🧱 1. HTML

В этом разделе ты изучаешь HTML: структуру страницы, семантику, формы, таблицы, медиа и встроенные возможности платформы.

Фокус раздела:

- семантика и структура;
- формы и медиа;
- нативные UI-паттерны платформы;
- HTML-контракты для SSR, загрузки и устойчивого рендера;
- командный quality bar для разметки.

Несмотря на это, здесь будут встречаться темы, которые вы еще не изучали: обучение нелинейно и требует возврата к предыдущим слоям.
Вы будете постоянно возвращаться назад, чтобы освежить знания. Это нормально.

Глубокие темы безопасности и производительности не являются ядром HTML-трека, но для `Senior+` здесь остаются целевые deep dive-уроки по platform constraints и review-risks. Полный доменный слой все равно остается у владельцев: [11. Безопасность](../11-безопасность/README.md) и [13. Performance](../13-performance/README.md).

## Как учиться в этом разделе

- Идти по ядру: **Нулевой блок -> Стажер -> Junior -> Middle -> Senior**.
- Держать в голове правило: HTML — это контракт структуры, доступности, загрузки и деградации, а не просто "теги ради тегов".
- Если тема уходит в глубокий `security` или `performance`, использовать HTML-урок как bridge и дальше переходить в владельца домена: [11. Безопасность](../11-безопасность/README.md) или [13. Performance](../13-performance/README.md).
- Для полного browser pipeline (`network -> parse -> execute -> render`) использовать сквозной урок: [4.0 Браузер: от URL до пикселя](../04-dom-и-browser/30-middle/4.0-браузер-от-url-до-пикселя-network-fetch-render.md).

## ✅ Ожидания по уровням

| Уровень | Что должен уметь |
|---|---|
| Нулевой блок | Уверенно пользоваться IDE и DevTools для быстрой и безопасной работы с HTML-кодом. |
| Стажер | Собирать корректный HTML-каркас, понимать базовую семантику, формы и таблицы. |
| Junior | Делать production-разметку: адаптивные медиа, формы, безопасные ссылки, machine-readable metadata, доступные таблицы. |
| Middle | Проектировать устойчивую разметку сложных страниц, выбирать нативные HTML-паттерны, писать SSR-safe и hydration-safe markup, понимать trade-offs по загрузке. |
| Senior / Senior+ | Формировать quality bar команды, видеть review-сигналы в HTML, разводить ownership между HTML, безопасностью и производительностью. Для Senior+ - уметь заходить в целевые deep dive по security/platform constraints, не теряя доменные границы. |
