# 🧱 1. HTML

В этом разделе ты разбираешь HTML: как собирать страницу, выбирать подходящие элементы, делать формы, таблицы и медиа так, чтобы разметка была понятной, устойчивой и полезной для реального интерфейса.

> 🚧 🚧 🚧
>
> Раздел в развитии. Состав уроков и глубина тем будут еще уточняться.
>
> 🚧 🚧 🚧

## Что здесь важно

- семантика и структура страницы;
- формы, таблицы и медиа;
- встроенные возможности браузера без лишнего JavaScript;
- разметка, которая не мешает загрузке и дальнейшей работе интерфейса;
- понятные требования к качеству HTML в команде.

Если по пути встретятся темы, которые ты еще не проходил, это нормально. Обучение во frontend не идет строго по прямой: иногда нужно вернуться назад, чтобы укрепить базу, а иногда заглянуть вперед, чтобы понять, зачем тема вообще нужна.

Глубокие темы по безопасности и производительности не являются ядром этого раздела, но для `Senior+` здесь остаются отдельные уроки про ограничения платформы и риски, которые важно замечать на ревью. Полная картина по этим темам вынесена в [11. Безопасность](../11-безопасность/README.md) и [13. Performance](../13-performance/README.md).

## Как проходить раздел

- Иди по порядку: **Нулевой блок -> Стажер -> Junior -> Middle -> Senior**.
- Держи в голове простое правило: HTML — это не просто теги, а основа структуры, доступности, загрузки и предсказуемого поведения страницы.
- Если тема уходит глубоко в безопасность или производительность, используй HTML как точку входа и дальше переходи в [11. Безопасность](../11-безопасность/README.md) или [13. Performance](../13-performance/README.md).
- Для полного browser pipeline (`network -> parse -> execute -> render`) использовать сквозной урок: [4.0 Браузер: от URL до пикселя](../04-dom-и-browser/30-middle/4.0-браузер-от-url-до-пикселя-network-fetch-render.md).

## ✅ Ожидания по уровням

| Уровень | Что должен уметь |
|---|---|
| Нулевой блок | Уверенно пользоваться IDE и DevTools для быстрой и безопасной работы с HTML-кодом. |
| Стажер | Собирать корректный HTML-каркас, понимать базовую семантику, формы и таблицы. |
| Junior | Делать production-разметку: адаптивные медиа, формы, безопасные ссылки, machine-readable metadata, доступные таблицы. |
| Middle | Проектировать устойчивую разметку сложных страниц, выбирать нативные HTML-паттерны, писать разметку, безопасную для SSR и hydration, и понимать, чем одни решения по загрузке лучше других. |
| Senior / Senior+ | Формировать требования к качеству HTML в команде, замечать риски в разметке и понимать, где заканчивается HTML и начинаются задачи безопасности и производительности. Для Senior+ - при необходимости уверенно заходить в углубленные темы по этим областям. |
