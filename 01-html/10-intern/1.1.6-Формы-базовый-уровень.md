# 1.1.6 Формы: базовый уровень

Форма — это место, где пользователь совершает действие: регистрация, покупка, запрос.

Плохая форма = потерянная конверсия.

---

## 🎯 Что ты освоишь

- связку `label` + поле ввода;
- базовые поля: `input`, `textarea`, `select`;
- правильные типы кнопок.

---

## Рабочий пример

```html
<form action="/signup" method="post">
  <label for="email">Email</label>
  <input id="email" name="email" type="email" required />

  <label for="about">О себе</label>
  <textarea id="about" name="about"></textarea>

  <button type="submit">Создать аккаунт</button>
</form>
```

---

## Что важно понять

- Поле без `name` не отправит значение на сервер.
- Поле без `label` хуже для доступности и UX. Если нужен более человеческий контекст, для кого это важно, см. [Семантика, скринридер и встроенные HTML API](./1.1.3-Семантика-скринридер-и-встроенные-API.md#для-кого-вообще-нужна-доступность).
- `button` без `type` внутри формы ведёт себя как `submit`.

---

## Типичные ошибки

1. Подпись поля делают только через placeholder.
1. Для всех кнопок ставят `type="button"` или вообще не ставят.
1. Поля формы не объединены логически.

---

## 🔧 Практика

1. Сверстай форму регистрации на 5 полей.
1. Для каждого поля сделай явный `label`.
1. Проверь форму только клавиатурой.

---

## ✅ Чеклист

- [ ] У каждого поля есть подпись
- [ ] У полей есть `name`
- [ ] У кнопок задан корректный `type`
- [ ] Формой можно пользоваться с клавиатуры
