# 1.3.2 Web Components: template, slot, shadow DOM

Этот урок не про «везде писать Web Components», а про понимание платформы и зрелый выбор инструментов.

---

## 🎯 Что ты освоишь

- роли `template`, `slot`, Custom Elements, Shadow DOM;
- когда Web Components полезны, а когда нет;
- какие риски по поддержке и интеграции стоит учитывать заранее.

---

## Концептуально

- `template` хранит разметку, пока ты её не клонируешь.
- `slot` даёт точки вставки внешнего контента.
- Shadow DOM изолирует внутреннюю структуру и стили.

```html
<template id="card-template">
  <article class="card">
    <slot name="title"></slot>
    <slot></slot>
  </article>
</template>
```

---

## Когда Web Components оправданы

- дизайн-система для нескольких продуктов;
- независимые виджеты, которые встраиваются в разные приложения;
- долгосрочная ставка на платформенные API.

## Когда лучше не усложнять

- короткий MVP;
- команда не готова поддерживать инфраструктуру и контракты компонентов;
- жёсткая привязка к одному фреймворку и его экосистеме.

---

## 🔧 Практика

1. Выдели reusable UI-блок и опиши его API (props/slots/events).
2. Сделай прототип на `template + slot`.
3. Сравни стоимость поддержки с эквивалентом во фреймворке.
