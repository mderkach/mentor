# 1.2.3 Формы: атрибуты и валидация

Зрелый джун не ждёт JavaScript для каждой проверки. Он умеет вытаскивать максимум из нативной HTML-валидации.

---

## 🎯 Что ты освоишь

- `required`, `minlength`, `maxlength`, `min`, `max`, `pattern`;
- разницу `readonly` и `disabled`;
- `autocomplete` и `inputmode` для удобства пользователя.

---

## Пример

```html
<label for="phone">Телефон (обязательное)</label>
<input
  id="phone"
  name="phone"
  type="tel"
  required
  pattern="[0-9+\-\s()]{10,}"
  autocomplete="tel"
  inputmode="tel"
/>
```

---

## Объяснение простыми словами

- `required` — нельзя отправить пустым.
- `pattern` — значение должно подходить под шаблон.
- `disabled` — поле выключено и не отправляется.
- `readonly` — поле видно и отправляется, но редактировать нельзя.

---

## Что важно для UX

- Не прячь обязательность только в звёздочку.
- Сообщения об ошибках должны быть человеческими.
- Тип поля должен соответствовать данным (`email`, `url`, `tel`).

---

## 🔧 Практика

1. Собери форму заявки с 6 полями.
2. Добавь нативные ограничения.
3. Проверь сценарии ошибок: пустое поле, короткое значение, неверный формат.

---

## ✅ Чеклист

- [ ] Валидация работает до отправки
- [ ] У полей корректные типы
- [ ] Ограничения (`min/max/length`) заданы осознанно
- [ ] Ошибки понятны пользователю
